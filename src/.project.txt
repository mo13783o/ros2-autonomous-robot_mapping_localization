# ROS2 + ESP32 Differential Drive Robot – Full System Context (With IMU Integration)

## Overview

I am building a real differential drive mobile robot using ROS2 and an ESP32-S3 microcontroller. The system is partially implemented and working. Development must be incremental and must NOT break existing functionality.

---

## Current System Architecture

### Low-Level (ESP32-S3)

* Reads quadrature encoders (left and right wheels) using interrupts
* Sends encoder counts over serial:

  ```
  ENC <left_counts> <right_counts>
  ```
* Receives motor commands from PC:

  ```
  <right_power> <left_power>
  ```
* Converts commands to hoverboard control (speed + steer)
* Uses **open-loop power control (NO PID yet)**

---

### ROS2 Side

#### Encoder Odometry Node

* Reads serial encoder data
* Computes differential drive odometry:

  * position (x, y, theta)
  * velocity (linear, angular)
* Publishes:

  ```
  /odom (nav_msgs/Odometry)
  ```
* Broadcasts TF:

  ```
  odom → base_link
  ```

---

## IMU Mounting

* IMU is mounted flat (horizontal) on the robot body
* Orientation:

  ```
  X → forward
  Y → left
  Z → up
  ```

This means:

```
Yaw rotation = rotation around Z axis
→ use gyro_z (gz)
```

## Current Goal (IMU Integration)

### Step 1 — Read IMU on ESP32

Read the following:

* Gyroscope:

  ```
  gx, gy, gz  (rad/s)
  ```
* Accelerometer:

  ```
  ax, ay, az  (m/s²)
  ```

---

### Step 2 — Calibration (CRITICAL)

* Measure gyro bias while robot is stationary
* Compute average:

  ```
  bias_z = average(gz)
  ```
* Apply correction:

  ```
  gz_corrected = gz - bias_z
  ```

---

### Step 3 — Serial Output Format

Extend current encoder message to include IMU:

Example:

```
ENC <L> <R> IMU <gx> <gy> <gz> <ax> <ay> <az>
```

OR minimal version:

```
ENC <L> <R> IMU <gz>
```

---

### Step 4 — ROS2 IMU Node

Create a node that:

* Reads IMU data from serial
* Publishes:

  ```
  /imu (sensor_msgs/Imu)
  ```

---

### Required Fields

```
angular_velocity
linear_acceleration
orientation (optional or identity if not available)
```

---

### Units (VERY IMPORTANT)

```
gyro → rad/s
accel → m/s²
```

If sensor outputs degrees:

```
rad/s = deg/s × π / 180
```

---

### Frame

```
frame_id = "imu_link"
```

---

## TF Setup

Add static transform:

```

```

Example:

```
x=0, y=0, z=0.1
```

## Why IMU Is Needed

* Encoder alone → heading drift
* IMU provides:

  ```
  angular velocity (gz)
  ```
* EKF combines:

  ```
  encoder + imu → better yaw estimate
  ```
